## 8. Data-subject rights & how they are met
- **Access / portability:** provided on request to [support email].
- **Rectification:** users can edit account details and documents in-app.
- **Erasure:** users may request deletion via the contact form or [support
  email]; an admin permanently deletes the user and all associated data
  (generations, files, feedback, credits, contact messages) using the
  "Erase data" action. Subject to any data we must retain by law.
- **Restriction / objection / withdraw consent:** handled on request; published
  testimonials are removed on request.

## 9. Security measures
- Passwords hashed (PBKDF2-SHA256); no plaintext passwords stored.
- Account-scoped access to documents; downloads via expiring links.
- Uploaded files parsed then deleted from disk immediately.
- Signed Stripe webhooks; authenticated admin portal.
- Authenticated pages sent with `Cache-Control: no-store`; `X-Content-Type-Options`
  and `Referrer-Policy` headers set.
- [At deployment: HTTPS/TLS, secure + HttpOnly cookies, CSP and X-Frame-Options
  at the reverse proxy, database backups, access controls on the server.]
